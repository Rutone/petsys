# ШУТИС Санал хураалын систем — Шаардлагын шийдвэрүүд

> 62 асуултын хариултаар тогтсон шийдвэрүүд. Хэрэгжүүлэлтийн эх сурвалж.

## Хугацаа, нэвтрүүлэлт (q59–62)

- **2026 оны 7-р сарын эхэнд** — зөвхөн **багш, ажилчдад** зориулсан бэлэн хувилбар (pilot).
- **2026 оны 9-р сард** — оюутан, багш, ажилтан **бүгд** нэвтрэх.
- Эхэндээ **Vercel** дээр deploy, дараа нь сургуулийн серверт шилжих төлөвтэй.
- Хэрэглэгчийн тоо: pilot үед ~**1000–1500** (багш, ажилчид). SQLite хангалттай; өсвөл PostgreSQL руу шилжинэ.

## Хэрэглэгч ба бүртгэл (q1–10)

- 3 төрлийн хэрэглэгч: **STUDENT / TEACHER / STAFF**. Тус бүр **тусдаа код** (оюутны/багшийн/ажилтны код).
- Кодын формат одоогоор **чөлөөтэй** (дараа сургуулийн системтэй тулгаж шалгана). Оруулсанд итгэнэ.
- Имэйл **ямар ч** байж болно. Нууц үг **ямар ч** урт/нийлмэл (шаардлага хассан).
- **Нууц үг сэргээх** (forgot password) урсгал хэрэгтэй.
- Профайл засах: **нэр, нууц үг, зураг** бүгд.
- Бүртгэл устгавал тухайн хүний **гарын үсэг, санал, сэтгэгдэл бүгд устана** (cascade).
- Зөвхөн **идэвхтэй** (active) оюутан/багш/ажилтан хамаарна → `isActive` талбар.
- Профайл дээр **сургууль/тэнхим, курс** мэдээлэл.

## Эрх, роль (q11–14)

- 3 роль: **USER / MODERATOR / ADMIN**.
  - **MODERATOR** — хэрэглэгчийн илгээсэн өргөдөл/санал асуулгыг **шүүх** (approve/reject).
  - **ADMIN** — бүхнийг хянана: хэн хэзээ санал өгсөн/буцаасан, хэн юуг reject хийсэн (**audit log**), хэрэглэгч удирдах, бусдыг admin/moderator болгох.
- **Албан байгууллагын баталгаажсан профайл/badge** (Оюутны зөвлөл, клуб, тэнхим) — байгууллагын нэрээр ✓ badge-тэй нийтлэх.
- **Scoped admin** (сургууль/тэнхимээр хязгаарлагдсан) — талбар бэлдэнэ, дараа гүйцээнэ.
- **SLA**: moderator/admin хэрэглэгчийн илгээсэн саналд **72 цагийн дотор** хариу өгөх ёстой; хэтэрсэнийг самбарт онцолно.

## Өргөдөл (q15–26)

- **Категори** — санал болгосон жагсаалтаар (Сургалт, Дотуур байр, Дэд бүтэц, Үйлчилгээ, Цалин хөлс/Хөдөлмөр, Бусад). Админ удирдана.
- **Файл/зураг хавсаргах** — size limit-тэй (жишээ: зураг ≤ 5MB).
- Зохиогч засвар хийвэл → дахин **moderator approve** шаардлагатай.
- Зохиогч **гарын үсэг цуглахаас өмнө** л устгаж болно; дараа нь зөвхөн admin.
- Гарын үсэг зурсан хүмүүсийн **зөвхөн тоо** харагдана (нэр нь нууц).
- Гарын үсэг зурахдаа **шалтгаан бичих хэрэггүй** (зүгээр л дэмжих/буцаах = like/unlike зарчим).
- **Гарын үсгээ буцаах** боломжтой (unsign).
- **Албан хариу** — захиргаа/хариуцсан алба өргөдөлд хариу бичих хэсэг. Хурдан шуурхай хариу чухал.
- **Статусын урсгал**: PENDING (хүлээгдэж буй) → APPROVED (идэвхтэй) / REJECTED (татгалзсан, шалтгаантай) → DISCUSSING (хэлэлцэж байгаа) → IN_PROGRESS (хэрэгжиж байгаа) → RESOLVED (шийдвэрлэсэн) / CLOSED (хаагдсан). Хэрэглэгчид ойлгомжтой харагдана.
- **Давхардсан өргөдлийг холбох** ("Энэ санал өмнө дурдагдсан") — relatedTo холбоос.
- Нээлттэй болсон өргөдөлд **үлдсэн хугацаа** ба **хэдэн гарын үсэг хэрэгтэй** нь харагдана.

## Санал асуулга (q27–33)

- Multi-select **үгүй**, survey **үгүй**, албан сонгуулийн горим **үгүй**, үр дүн нуух/саналаа өөрчлөх **үгүй**.
- Санал **нууц** — admin харж чадна, гэхдээ **маш нууц** түвшинд хадгална.
- **Audience targeting**: багшийн санал асуулга зөвхөн багшид, ажилтных ажилтанд, оюутных оюутанд харагдана (ALL / STUDENT / TEACHER / STAFF).

## Сэтгэгдэл (q34–38)

- Зөвхөн **Like товч**. Reply/report/filter/edit **байхгүй**.

## Админ, модераци (q39–46)

- Татгалзсан өргөдлийг **зассаны дараа дахин илгээх** (resubmit).
- Нийтлэгдсэн өргөдөл/асуулгыг admin **буцаан нуух/хүчингүй** болгох.
- Хэрэглэгчийг **блоклох/бан**.
- **Audit log** бүртгэх.
- Админ самбарт **хэрэглэгчдийн жагсаалт, хайлт, удирдлага**.
- Статистик/үр дүнг **Excel/CSV** татах.
- Шинэ өргөдөл/асуулга ирэхэд **админд имэйл мэдэгдэл**.

## Мэдэгдэл (q47–49)

- Хэрэглэгчид **имэйл мэдэгдэл** (зөвшөөрсөн/татгалзсан, зорилтод хүрсэн г.м.).
- Сайт доторх **мэдэгдлийн төв (🔔)**.
- Имэйл **манай системийн SMTP**-ээр илгээнэ.

## Аюулгүй байдал (q50–53)

- **Rate limiting** — тийм (нэг IP-ээс олон бүртгэл/хүсэлт хязгаарлах).
- CAPTCHA **үгүй**.
- Өөр хүний кодоор бүртгэх эрсдэл байгааг хүлээн зөвшөөрнө, **одоохондоо алгасна**.
- **Нууцлалын бодлого / үйлчилгээний нөхцөл** хуудас.

## UI/UX (q54–58)

- **Dark mode** — тийм.
- Англи хувилбар **үгүй** (эхэндээ зөвхөн Монгол).
- Зөвхөн **веб** (mobile app байхгүй, responsive).
- **ШУТИС-ийн лого, албан өнгө** ашиглана. Front-end-ийг оюутны вебээс (student.must.edu.mn) санаа авч **sidebar layout, цэвэрхэн картан** хэлбэртэй болгоно.
- Нүүр хуудас: **их саналтай / хамгийн сүүл / хамгийн эхэнд оруулсан** sort болон **хайлт**.
